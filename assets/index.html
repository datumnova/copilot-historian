How Copilot Historian works

One binary, zero prerequisites. The whole dashboard — scanner, search index, HTTP server and this UI — ships inside a single ~1 MB executable. Nothing is installed, nothing is written to disk, nothing leaves your machine.
Where the numbers come from. Each CLI run appends a session.shutdown event per run segment. Historian sums those segments: token details, premium requests, totalNanoAiu, per-model metrics, and code changes (lines ± and files touched). Sessions that never shut down cleanly show "—".
Premium vs AIU. Premium requests are the billing unit (flat per user request, weighted by model multiplier). AIU — AI Units — measure the compute actually consumed, scaling with tokens, model tier and reasoning effort. Comparing both tells you which models burn hottest per request.
Rescan re-reads the directory in place and atomically swaps the in-memory snapshot — new and still-running sessions appear without restarting.
